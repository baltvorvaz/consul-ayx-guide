# consul-ayx-guide

一个用于归档和发布独立 HTML 页面的仓库。

## 项目简介

本项目主要用于存放和发布独立的 HTML 页面资源。这些页面不针对任何特定域名或网站，仅作为通用页面归档使用。适合需要临时托管静态页面、展示示例或保存历史版本的场景。

## 目录结构

```
consul-ayx-guide/
├── index.html          # 主入口页面
├── pages/              # 存放独立 HTML 页面
│   ├── example1.html
│   ├── example2.html
│   └── ...
├── assets/             # 静态资源（CSS、JS、图片等）
│   ├── css/
│   ├── js/
│   └── images/
└── README.md           # 本文件
```

- **pages/**：存放所有独立的 HTML 页面文件。
- **assets/**：存放页面所需的静态资源，按类型归类。
- **index.html**：可选的入口页面，用于导航或说明。

## 页面归档说明

- 每个 HTML 页面均为独立文件，不依赖外部域名或特定环境。
- 页面资源尽量使用相对路径引用，确保迁移或离线查看时可用。
- 归档页面可能包含示例数据、演示内容或文档片段，请根据实际需要自行调整。

## 维护说明

- 如需添加新页面，请在 `pages/` 目录下创建对应的 HTML 文件。
- 若页面使用外部资源（如字体、库文件），建议下载到 `assets/` 目录中引用。
- 请保持目录结构清晰，避免重复或冗余文件。
- 本仓库不保证页面内容的实时更新或长期可用性。

## 使用方式

1. 克隆或下载本仓库。
2. 直接在浏览器中打开 `index.html` 或 `pages/` 下的任意页面。
3. 也可通过 GitHub Pages 或其他静态托管服务发布。
